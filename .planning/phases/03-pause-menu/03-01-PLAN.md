# Plan 03-01: Pause Menu — ESC Overlay with Resume / Restart / Home

## Goal

ESC key during a live match freezes Matter.js physics, stops the UI timer and
cooldown updates, and shows a `PauseScene` overlay with three buttons: Resume,
Restart Match, and Return to Menu. A second ESC press resumes the game. Each
button navigates correctly and cleans up all scenes.

## Context

`GameScene` has no pause handling today. `UIScene.update()` runs every frame and
decrements the timer regardless. Matter.js continues simulating while the tab is
in focus. Players need a way to pause, check the score, and navigate away without
abandoning a match irreversibly.

The cleanest Phaser pattern for a pause overlay is `this.scene.launch('PauseScene',
{ ... })` — it runs PauseScene on top of GameScene and UIScene without stopping
their scene lifecycle, but we manually pause physics and skip UI updates via a
`this.paused` flag on `GameScene`.

`GameScene.init()` already stores `p1CharId`, `p2CharId`, and `vsMode` — these are
passed through to PauseScene so "Restart Match" can relaunch with the same config.

## Must Haves

- [ ] ESC key during GameScene freezes Matter.js physics (`matter.world.pause()`)
- [ ] `GameScene.paused` flag is set so UIScene skips its timer and cooldown updates
- [ ] PauseScene launches as an overlay (not replacing GameScene)
- [ ] PauseScene shows: "PAUSED" title, Resume button, Restart Match button, Return to Menu button
- [ ] "Resume" unpauses physics, clears the flag, stops PauseScene
- [ ] Second ESC press while paused also resumes (same as Resume button)
- [ ] "Restart Match" stops PauseScene + UIScene, restarts GameScene with same init data
- [ ] "Return to Menu" stops PauseScene + UIScene + GameScene, starts MenuScene
- [ ] Timer and cooldowns do not advance while paused

## Tasks

### Task 1: Create `PauseScene`

**File:** `src/scenes/PauseScene.js` (new file)

**Action:** Create a new Phaser scene that renders a semi-transparent overlay,
a "PAUSED" heading, and three interactive buttons. It communicates back to
GameScene via `this.scene.get('GameScene')` so it does not need GameScene
stored as a closure.

**Details:**

```js
import Phaser from 'phaser';

export class PauseScene extends Phaser.Scene {
  constructor() { super({ key: 'PauseScene' }); }

  init(data) {
    // Store init data for Restart so we can relaunch GameScene identically
    this.initData = data; // { p1CharId, p2CharId, vsMode }
  }

  create() {
    const W = this.scale.width;
    const H = this.scale.height;

    // Dim overlay — covers GameScene + UIScene visually
    this.add.rectangle(W / 2, H / 2, W, H, 0x000000, 0.55).setDepth(0);

    // Title
    this.add.text(W / 2, H / 2 - 120, 'PAUSED', {
      fontSize: '72px',
      fontFamily: 'Arial Black, sans-serif',
      color: '#ffffff',
      stroke: '#000000',
      strokeThickness: 8,
    }).setOrigin(0.5).setDepth(1);

    // Button helper
    const makeButton = (y, label, callback) => {
      const btn = this.add.text(W / 2, y, label, {
        fontSize: '32px',
        fontFamily: 'Arial Black, sans-serif',
        color: '#ffffff',
        backgroundColor: '#1a4a8a',
        padding: { x: 32, y: 12 },
      })
        .setOrigin(0.5)
        .setInteractive({ useHandCursor: true })
        .setDepth(1);

      btn.on('pointerover', () => btn.setStyle({ color: '#ffdd44' }));
      btn.on('pointerout',  () => btn.setStyle({ color: '#ffffff' }));
      btn.on('pointerdown', callback);
      return btn;
    };

    makeButton(H / 2 - 20, 'Resume',          () => this._resume());
    makeButton(H / 2 + 70, 'Restart Match',   () => this._restart());
    makeButton(H / 2 + 150, 'Return to Menu', () => this._menu());

    // ESC also resumes
    this.input.keyboard.on('keydown-ESC', () => this._resume());
  }

  _resume() {
    const gs = this.scene.get('GameScene');
    gs.matter.world.resume();
    gs.paused = false;
    this.scene.stop('PauseScene');
  }

  _restart() {
    const data = this.initData;
    this.scene.stop('PauseScene');
    this.scene.stop('UIScene');
    // GameScene.init() re-runs automatically on scene.restart()
    this.scene.get('GameScene').scene.restart(data);
  }

  _menu() {
    this.scene.stop('PauseScene');
    this.scene.stop('UIScene');
    this.scene.stop('GameScene');
    this.scene.start('MenuScene');
  }
}
```

Key implementation notes:
- `this.scene.get('GameScene')` always resolves the live GameScene instance —
  no need to pass it through `init()` data.
- `scene.restart(data)` relaunches GameScene with the same `p1CharId`, `p2CharId`,
  `vsMode` via `init(data)` — GameScene.init() resets score, timer, and flags.
- Stopping UIScene before `scene.restart()` prevents a duplicate UIScene launch
  because GameScene.create() calls `this.scene.launch('UIScene', ...)` again.

### Task 2: Wire ESC pause into `GameScene` and guard `UIScene`

**Files:** `src/scenes/GameScene.js`, `src/scenes/UIScene.js`, `src/main.js`

**Action:** Register the ESC key in `GameScene.create()`, add the `paused` flag
guard to `GameScene.update()`, guard `UIScene.update()` against the paused flag,
and register `PauseScene` in `src/main.js`.

**Details:**

**2a — `src/main.js`: register PauseScene**

Add the import at the top of the file alongside the other scene imports:

```js
import { PauseScene } from './scenes/PauseScene.js';
```

Add `PauseScene` to the `scene` array in the Phaser config:

```js
scene: [BootScene, MenuScene, GameScene, UIScene, PauseScene, ResultScene],
```

Order matters for initial boot only — PauseScene is launched on demand, so
position in the array is not critical beyond it being registered.

**2b — `src/scenes/GameScene.js`: ESC listener + paused flag**

In `GameScene.init()`, initialise the flag so it resets correctly on restart:

```js
this.paused = false;
```

In `GameScene.create()`, after the keyboard `addKeys()` block (after line 40),
add the ESC listener:

```js
this.escKey = this.input.keyboard.addKey(Phaser.Input.Keyboard.KeyCodes.ESC);
this.escKey.on('down', () => {
  if (this.matchOver) return;
  if (this.paused) {
    // Already paused — ESC resumes (mirrors PauseScene._resume())
    this.matter.world.resume();
    this.paused = false;
    this.scene.stop('PauseScene');
  } else {
    this.matter.world.pause();
    this.paused = true;
    this.scene.launch('PauseScene', {
      p1CharId: this.p1CharId,
      p2CharId: this.p2CharId,
      vsMode: this.vsMode,
    });
  }
});
```

In `GameScene.update()`, add a paused guard at the top (after the existing
`if (this.matchOver) return;` check):

```js
if (this.paused) return;
```

This prevents `ball.update()` and player updates from running while physics is
frozen, avoiding any desync between the physics world and game state.

**2c — `src/scenes/UIScene.js`: skip updates while paused**

In `UIScene.update()`, extend the existing early-return guard on line 74 from:

```js
if (!gs || gs.matchOver) return;
```

to:

```js
if (!gs || gs.matchOver || gs.paused) return;
```

This single-line change stops the timer display from counting down and the
cooldown bars from animating while the game is paused. Score display is also
skipped — acceptable because the score cannot change while paused.

## Verification

1. Run `npm run dev` and start a match.
2. Press ESC — physics should freeze immediately (ball stops mid-air, players
   stop moving). The PauseScene overlay appears with "PAUSED" and three buttons.
   Confirm the timer on the HUD does not advance.
3. Click "Resume" (or press ESC again) — physics resumes, players can move, timer
   continues from where it stopped.
4. Pause again, then click "Restart Match" — the match resets with the same two
   characters and vs mode. Score resets to 0:0 and timer resets to 1:30.
5. Pause again, then click "Return to Menu" — all game scenes stop cleanly and the
   main menu is displayed. No leftover overlays or ghost scenes.
6. Verify that pressing ESC after a match ends (matchOver = true) does nothing —
   the pause listener is guarded by `if (this.matchOver) return`.
7. Verify that ability cooldown bars are frozen during pause by activating an
   ability just before pausing — the bar fill should not change while paused.
