# Plan 02-01: Head Visuals — Larger Sprites + Squash-and-Stretch Wobble

## Goal

Increase the displayed head sprite size from 96px to 128px and add a
squash-and-stretch scale tween that triggers on jump landing, ball contact,
and ability use. Physics collision bodies are untouched — only the visual
layer changes.

## Context

Currently `HEAD_DISPLAY_SIZE = PLAYER.headRadius * 2 + 16 = 96px`. The heads
read as small on a 1280×720 canvas. Increasing the display size to 128px makes
characters more readable without touching the 40px physics circle. Adding
spring-style scale tweens on impact events makes the game feel juicy and alive,
a core trait of the Head Soccer genre.

The physics body is set via `setCircle(PLAYER.headRadius, ...)` independently of
the sprite scale, so enlarging the display scale does not affect collisions.
`setFixedRotation()` is already applied, so wobble must be purely scale-based
(no rotation involved).

## Must Haves

- [ ] Head sprites render at ~128px display size (larger than current 96px)
- [ ] Physics circle (headRadius 40) is unchanged — collision body is unaffected
- [ ] On jump landing (airborne → grounded transition), head squishes then bounces back
- [ ] On ball contact, head squishes then bounces back
- [ ] On ability use, head squishes then bounces back
- [ ] Wobble decays fully within ~300 ms — no perpetual oscillation
- [ ] Both P1 and P2 heads wobble independently

### Task 2: Add `_wobble()` helper method

**File:** `src/entities/Player.js`

**Action:** Add a private `_wobble()` method to the `Player` class that fires a
squash-and-stretch scale tween on `this.sprite`. Guard against overlapping tweens
with a debounce flag.

**Details:**

Add the following method to the `Player` class (place it after `_emitParticles`):

```js
_wobble() {
  // Prevent stacking tweens — if one is already running, skip
  if (this._wobbling) return;
  this._wobbling = true;

  // Capture the sprite's current base scale (set during _create)
  const base = this.sprite.scaleX; // scaleX === scaleY when no flip involved

  this.scene.tweens.add({
    targets: this.sprite,
    scaleX: base * 1.15,   // stretch horizontally
    scaleY: base * 0.85,   // squash vertically
    duration: 60,
    ease: 'Sine.easeOut',
    yoyo: true,            // bounce back to base automatically
    onComplete: () => {
      // Restore exact base scale (yoyo lands close but pin precisely)
      this.sprite.setScale(base);
      this._wobbling = false;
    },
  });
}
```

Notes:
- `yoyo: true` makes Phaser reverse the tween automatically — no second tween needed.
- Total wobble duration is 60 ms × 2 (yoyo) = ~120 ms visible motion, which decays
  naturally to base scale.
- `base` is captured at call time so it always uses the correct display scale even
  if `HEAD_DISPLAY_SIZE` changes later.
- `_wobbling` is initialised implicitly as `undefined` (falsy) — no constructor
  change is required.

### Task 3: Wire wobble triggers

**File:** `src/entities/Player.js`

**Action:** Call `this._wobble()` at the three trigger points: jump landing, ball
contact, and ability use. Also register the Matter.js collision listener in
`_create()` to detect ball contact.

**Details:**

**3a — Jump landing (in `update()`):**

`checkGround()` is called every frame and sets `this.isOnGround`. Track the
previous ground state and fire wobble on the false→true transition:

In `update()`, replace the existing `this.checkGround();` call (line 57) with:

```js
const wasOnGround = this.isOnGround;
this.checkGround();
if (!wasOnGround && this.isOnGround) {
  this._wobble();
}
```

**3b — Ball contact (collision listener in `_create()`):**

After the `this.sprite.setDepth(20);` line in `_create()`, add:

```js
this.scene.matter.world.on('collisionstart', (event) => {
  event.pairs.forEach(pair => {
    const { bodyA, bodyB } = pair;
    const bodies = [bodyA, bodyB];
    const isSelf  = bodies.some(b => b === this.sprite.body);
    const isBall  = bodies.some(b => b.label === 'ball');
    if (isSelf && isBall) this._wobble();
  });
});
```

This is a per-Player listener. Both P1 and P2 register their own listener so
each wobbles independently on contact with the ball.

**3c — Ability use (in `_useAbility()`):**

At the very top of `_useAbility(time, ball)`, before the `id` switch, add:

```js
this._wobble();
```
